# Stone V2 Product Thesis

## Status

- `Verified`: This document defines the customer, promise, and market thesis for Stone V2.
- `Verified`: This document is doc-only. It does not authorize runtime edits by itself.

## Scope

- `Verified`: Goal is to define who Stone V2 is for and what it should promise.
- `Verified`: This document covers:
  - target customer
  - core problem
  - value proposition
  - differentiation
  - claims to avoid

## Primary customer

- `Inference`: Israeli Shopify developers
- `Inference`: small Shopify agencies
- `Inference`: freelancers building multiple client stores

## Secondary customer

- `Inference`: Hebrew-speaking merchants who later edit the store from the theme editor

## Problem statement

- `Inference`: Israeli developers often start from international LTR-first themes and then spend time fighting:
  - RTL layout issues
  - Hebrew typography and spacing issues
  - poor local trust and support defaults
  - English-only or foreign-feeling theme-editor experiences
  - limited editor flexibility that pushes ordinary changes into CSS or Liquid work

## Product thesis

- `Inference`: Stone V2 should exist to remove repeated RTL and Hebrew rescue work from Israeli Shopify implementation.
- `Inference`: The clearest thesis is:

  > Stone V2 is a premium RTL-native Shopify theme platform for Israeli developers, with deep editor-driven customization, bilingual Hebrew or English editor support, and enough reusable breadth to cover most client stores from one base theme.

## Value proposition for developers

- `Inference`: less RTL rescue work
- `Inference`: less routine custom CSS
- `Inference`: faster delivery across different industries
- `Inference`: stronger client handoff
- `Inference`: more consistent project reuse

## Value proposition for merchants

- `Inference`: more local storefront behavior
- `Inference`: easier content editing
- `Inference`: Hebrew-friendly theme-editor labels and guidance
- `Inference`: less dependence on ongoing developer help for ordinary content changes

## Differentiation pillars

### 1. RTL-native from the start

- `Inference`: V2 should be built for RTL first, not retrofitted later.

### 2. Bilingual theme-editor experience

- `Inference`: Hebrew and English schema coverage should be a major headline feature.
- `Inference`: This matters for both developer workflow and merchant handoff.

### 3. Developer-grade editor control

- `Inference`: The theme editor should be powerful enough that many standard client requests can be solved there before code is touched.

### 4. Multi-industry usability

- `Inference`: The theme should not feel locked to one niche. It should be reusable across multiple common Israeli Shopify industries.

### 5. Premium platform breadth

- `Inference`: V2 should feel broad enough that developers can think `this might be the only RTL base theme I need for most stores`.

## Marketing-safe claims

- `Inference`: `Built for Israeli Shopify developers`
- `Inference`: `Stop porting foreign themes to RTL`
- `Inference`: `Customize deeply from the editor before reaching for code`
- `Inference`: `Hebrew and English editor support`
- `Inference`: `Industry-ready presets so you start halfway done`

## Claims to avoid literally

- `Inference`: `No code ever`
- `Inference`: `Every possible store without exception`
- `Inference`: `Zero effort customization`
- `Inference`: `The last theme every developer in every scenario will ever need`

## Better promise language

- `Inference`: `The last RTL base theme most Israeli Shopify developers will need`
- `Inference`: `Built to reduce difficult theme work, not create more of it`
- `Inference`: `Code only when the project truly needs it`

## Final recommendation

- `Verified`: Market Stone V2 as a premium developer platform, not as just another RTL-compatible theme.
- `Inference`: The strongest promise is faster, more local, more editor-driven store delivery from one reusable base.

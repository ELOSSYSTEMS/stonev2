# Phase 4 Preset-Pack Runtime Boundary Risks v1

1. `Inference`: A preset-pack runtime boundary could become broad rollout if exact files are not stated.
2. `Inference`: Shared settings could become per-preset schema sprawl if not constrained.
3. `Inference`: Preset language could smuggle in search, cart, recommendation, or hardening behavior.
4. `Inference`: A global preset selector could be overstated as complete preset-pack realization even though host assignment and starter content remain blocked.
5. `Verified`: The mitigation is an exact five-file boundary, exact selector-only scope, explicit template/section/JavaScript exclusions, and explicit blocked-field ledger.

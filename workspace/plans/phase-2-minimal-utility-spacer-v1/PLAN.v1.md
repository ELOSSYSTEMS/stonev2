## Builder

- `Inference`: The verified shell, verified section shell, verified divider-band section, verified prompt section, and verified neutral template hosts already prove structural separation and contained prompting, so the next smallest unproven step is a bounded utility spacer that proves adjustable neutral spacing as a first-class utility-only surface.
- `Inference`: The future runtime slice should proceed in this order:
  1. Extend `project/assets/stone-base.css` with neutral spacer primitives that support bounded spacing options without family styling.
  2. Create `project/sections/stone-utility-spacer.liquid` as one utility-only section that consumes the existing shared section shell unchanged and renders one spacing surface only.
  3. Create `project/templates/page.stone-runtime-spacer.json` as one neutral host that mounts only the new section for isolated validation.
  4. Update `project/locales/he.default.schema.json` and `project/locales/en.schema.json` together so the new section stays bilingual and terminology-aligned.
- `Inference`: This path is minimal because it proves reusable spacing infrastructure without reopening the shared shell, adding runtime JS, adding block files, adding snippets, or composing multiple sections into a page-level experience.

- `Inference`: Another failure mode is false-proof drift if the spacer only restates behavior already covered by the divider-band slice instead of proving adjustable spacing as a distinct structural helper.
- `Inference`: A third failure mode is protected-surface backtracking if the spacer proof starts requiring edits to `project/snippets/stone-section-shell.liquid`, `project/layout/theme.liquid`, `project/assets/stone-runtime.js`, or `project/config/settings_schema.json`.

# Phase 4 PDP Capability Boundary Risks v1

1. `Inference`: Dedicated size chart and spec sheet sections could become preset-specific architecture if named or scoped by industry.
2. `Inference`: Spec sheet work could expand into recommendations, compatibility engines, or inventory behavior.
3. `Inference`: Size chart work could expand into sizing logic, fit prediction, or compliance-sensitive claims.
4. `Inference`: Preset assignment could happen before section capability exists.
5. `Verified`: The mitigation is shared section naming, docs-only runtime-boundary first, no preset assignment, no product data mutation, and no cart/checkout/recommendation behavior.

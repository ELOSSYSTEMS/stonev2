## Recommended Phase 3 Entry Sequence

### Phase 3A, merchandising foundation boundary

- `Verified`: First author a broader docs-only boundary for a merchandising foundation slice.
- `Inference`: That slice should likely authorize more than the old five-file protocol, because product-card behavior and merchandising structure cannot be represented honestly otherwise.
- `Verified`: The first broader boundary must define exact allowed files, preserved constraints, distinctness case, and explicit stop conditions.

### Phase 3B, product-card contract and shared merchandising primitives

- `Inference`: Establish one canonical product-card contract before any grid, rail, or comparison family expands.
- `Inference`: Normalize media ratio, metadata exposure, badges, CTA posture, and RTL-safe card spacing here rather than ad hoc in later sections.

### Phase 3C, first merchandising family proof

- `Inference`: After the product-card contract exists, implement one normalized merchandising section that proves category-first or curated-product structure without fragmenting file vocabulary.

### Phase 3D, collection/PDP integration follow-on

- `Inference`: Only after merchandising primitives stabilize should broader collection companion and PDP companion families absorb richer product-card behavior.

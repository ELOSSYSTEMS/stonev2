## Builder

- `Inference`: The verified shell, verified section shell, verified utility richtext section, verified utility action group section, verified utility key-value list section, verified utility divider band section, and verified neutral template hosts already prove neutral content hosting, bounded actions, structured information, and structural separation, so the next smallest unproven step is a bounded contained notice surface inside one utility-only section.
- `Inference`: The future runtime slice should proceed in this order:
  1. Extend `project/assets/stone-base.css` with neutral notice primitives that support restrained containment without family styling.
  2. Create `project/sections/stone-utility-notice.liquid` as one utility-only section that consumes the existing shared section shell unchanged and renders one contained neutral notice surface only.

- `Inference`: This path is minimal because it proves reusable contained-emphasis behavior without reopening the shared shell, adding runtime JS, adding block files, adding snippets, or composing multiple sections into a page-level experience.
